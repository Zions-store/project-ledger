# Project Docs

Initialize and maintain a three-document project knowledge system that gives AI agents persistent project context across sessions.

## What It Does

Creates and maintains three documents:

| File | Purpose | Update Frequency |
|------|---------|-----------------|
| `AGENTS.md` | Project overview: identity, directory structure, class architecture, entry points, dependencies, Notes | Per milestone (~3-5 new files/classes) |
| `PROJECT_STATE.md` | Living document: full class spectrum, blueprint config, input system map, game flows, done/todo list | Every session end |
| `DEVLOG.md` | Development log: timeline entries with what was done, why, and lessons learned | Per day or per feature milestone |

## Why Project Docs

Without persistent documentation, every new AI agent session starts from scratch. This skill creates a living documentation system that:

- **Reduces context preamble**: the agent reads `AGENTS.md` once and understands the project
- **Tracks progress**: `PROJECT_STATE.md` maintains a running todo/done list
- **Builds institutional memory**: `DEVLOG.md` records decisions and rationale

## Usage

Just tell your AI agent:

```
init project docs          (initialize three-document system)
project-docs               (update existing documents)
update project status      (update todo/done list in PROJECT_STATE.md)
建立文档体系                   (initialize in Chinese)
更新项目状态                   (update in Chinese)
```

Requires an existing `AGENTS.md` (generated by project-onboard or created manually).

## File Structure

```
project-docs/
├── SKILL.md                    # Skill definition
├── README.md                   # This file
├── CHANGELOG.md                # Version history
├── LICENSE                     # GPL-3.0
├── maintenance-spec.md         # Detailed update rules
└── templates/
    ├── DEVLOG.md.tmpl          # Shared across all project types
    ├── PROJECT_STATE.md.tmpl   # Generic template (Node, Python, Rust, Go, Java, C++, C#, Lua, etc.)
    ├── unreal/
    │   └── PROJECT_STATE.md.tmpl  # Unreal Engine template (Blueprint, InputActions, Game Systems, Hot Reload)
    └── unity/
        └── PROJECT_STATE.md.tmpl  # Unity template (Prefabs, Input System Package, Scenes, Build Profiles)
```

## Template Selection

The skill reads the `Type` field from `AGENTS.md` and auto-selects the matching template:

| AGENTS.md Type | Template |
|---|---|
| `unreal` | `templates/unreal/PROJECT_STATE.md.tmpl` |
| `unity` | `templates/unity/PROJECT_STATE.md.tmpl` |
| `nodejs`, `python`, `rust`, `go`, `java`, `cpp`, `csharp`, `lua`, `general` | `templates/PROJECT_STATE.md.tmpl` |
